## Overview

The HRMS Developer Portal provides a centralized hub for creating, managing, and monitoring your HRMS integrations. This guide walks you through the key features and workflows of the portal.

## Portal Navigation

The portal includes several key sections:
- **Dashboard** - Overview of your applications and activities
- **Apps** - Manage your HRMS integrations
- **Webhook Logs** - Monitor event deliveries
- **Demo Companies** - Test your integrations
- **Settings** - Configure your account

## Creating Applications

To create a new HRMS integration:

1. Click the "Create App" button in the top right corner
2. Fill in the basic information:
   - App Name
   - Description
   - Category
   - Support Email
   - Website URL
   - Privacy Settings
   - Webhook Security Level

## Managing Permissions

Each application can be granted specific permissions across different HRMS modules:

<CardGroup>
  <Card title="Dashboard Access" icon="gauge">

  </Card>
  <Card title="Employee Management" icon="users">
    Control over employee data and records
  </Card>
  <Card title="Role Management" icon="user-gear">
    Permissions for role assignments and modifications
  </Card>
  <Card title="Device Management" icon="mobile">
    Access to device registration and tracking
  </Card>
</CardGroup>

### Available Permission Scopes

- **Dashboard**: View and manage dashboard components
- **Role**: User role management
- **Device**: Device registration and tracking
- **Department**: Organizational structure management
- **Position**: Job position configurations
- **Accounting**: Financial data access
- **Employee**: Staff record management
- **Leave**: Time-off management
- **Entity**: Company entity control

## Security Best Practices

<AccordionGroup>
  <Accordion title="Webhook Security">
    - Use HTTPS endpoints
    - Implement signature verification
    - Set up retry policies
    - Monitor webhook logs
  </Accordion>
  
  <Accordion title="Permission Management">
    - Follow principle of least privilege
    - Regularly audit permissions
    - Revoke unused access
  </Accordion>
  
  <Accordion title="API Security">
    - Secure storage of credentials
    - Implement rate limiting
    - Use OAuth 2.0 for authentication
  </Accordion>
</AccordionGroup>
